# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Static marketing site for "Chape Expert", a French (Île-de-France) concrete-screed installation business. A home page (hero/services/projects/contact sections) plus SEO landing pages, conseils articles, réalisations, a `/devis` quote form, and a 404 — all content is in French, served SSG via `vite-ssg` and deployed to **Cloudflare Pages** (`wrangler.toml`, `pages_build_output_dir = "dist"`).

Built on the Vitesse template (Vue 3 + Vite + TypeScript).

## Commands

Package manager is **pnpm** (required — `pnpm-workspace.yaml` defines dependency catalogs). Node 18+.

- `pnpm dev` — dev server on port 3333 (auto-opens)
- `pnpm build` — production SSG build (`vite-ssg build`, outputs to `dist/`)
- `pnpm preview` — preview the built site
- `pnpm lint` — ESLint (antfu config). Auto-fix runs pre-commit via `simple-git-hooks` + `lint-staged`
- `pnpm typecheck` — `vue-tsc --noEmit`
- `pnpm test` / `pnpm test:unit` — Vitest (jsdom env, picks up `test/**/*.test.ts`)
- `pnpm test:e2e` — Cypress (interactive)
- `pnpm sizecheck` — bundle visualizer

No single-test helper script — run one file with `pnpm vitest run path/to/file.test.ts` or a name filter with `pnpm vitest -t "name"`.

## Architecture

### Build & routing — heavy reliance on auto-generation

Several files in `src/` are **generated** and must not be hand-edited:

- `src/auto-imports.d.ts` — from `unplugin-auto-import` (Vue/VueUse/vue-i18n/vue-router composables + `src/composables/` + `src/stores/` are globally available without imports)
- `src/components.d.ts` — from `unplugin-vue-components` (every `.vue` or `.md` file in `src/components/` auto-registers as a global component; reference `<Header />`, `<Hero />`, etc. without importing)
- `src/typed-router.d.ts` — from `unplugin-vue-router` (file-based routing over `src/pages/`; routes auto-generate from filenames, e.g. `[...all].vue` → catch-all 404)

Because of auto-import, new composables/stores/components placed in their respective dirs are usable immediately without touching import statements. Restart the dev server if autogenerated types go stale.

### Layout system

`vite-plugin-vue-layouts` wraps routes with layouts from `src/layouts/`. A page picks a layout via an SFC `<route>` block:

```vue
<route lang="yaml">
meta:
  layout: home
</route>
```

`index.vue` uses `home`, `[...all].vue` uses `404`, everything else falls back to `default.vue`.

### Module system

Any `src/modules/*.ts` exporting `install: UserModule` is auto-wired in `src/main.ts` via `import.meta.glob`. Current modules install i18n, Pinia, NProgress, and PWA. Add cross-cutting app setup here rather than editing `main.ts`.

### i18n

- Single active locale: `fr` (hardcoded default in `src/modules/i18n.ts`, loaded async).
- Translations live in `locales/*.yml` and are bundled via `@intlify/unplugin-vue-i18n`.
- **Note:** `locales/` currently contains only a README — the `not-found` key used in `[...all].vue` has no backing translation file. Add `locales/fr.yml` if you need real translations; component copy is otherwise hardcoded in French inside the Vue SFCs.

### Styling

Tailwind v4 (via `@tailwindcss/vite`) + daisyUI v5. The custom daisyUI theme `mytheme` is defined inline in [src/styles/main.css](src/styles/main.css) — tweak tokens there, not in a separate config. Primary color is lime green (`#cbf36e`), not the blue mentioned in the old README.

Three Google Fonts (Roboto/Alfabet/Inter) are imported at the top of `main.css` and exposed via utility classes `.title-text`, `.description-text`, `.sans-serif-text`.

### Animations

`@vueuse/motion` is registered globally via [src/modules/motion.ts](src/modules/motion.ts). Use `v-motion` with `:initial` + `:visible-once` (scroll-triggered, fires on enter) or `:enter` (fires on mount). Existing components use stagger via per-item `delay: 100 + i * 120` patterns.

### Path alias

`~/` → `src/` (set in both `vite.config.ts` and `tsconfig.json`).

### SSG + SEO specifics

- `pnpm build` runs `vite-ssg` which pre-renders routes, inlines critical CSS via Beasties, and calls `generateSitemap()` on finish (output: `dist/sitemap.xml`).
- **Dynamic routes are not pre-rendered automatically.** `ssgOptions.includedRoutes` in `vite.config.ts` enumerates the `/realisations/:slug` pages (plus `/404`) so they get static HTML and land in the sitemap. **Keep that slug list in sync with `src/composables/projects.ts`** when adding/removing a project, or the new page will be invisible to crawlers.
- Global `<head>` meta (OG, Twitter, geo, business) lives in `src/App.vue` via `useHead`. OG/Twitter image URLs must be absolute.
- Per-page head overrides go through `useHead` inside the page SFC (see `src/pages/index.vue`).
- JSON-LD: the home page uses `<StructuredData />` (LocalBusiness); landing/conseils/réalisations pages build schemas via helpers in [src/utils/landing-schema.ts](src/utils/landing-schema.ts) (`buildBreadcrumbSchema`, `buildFaqSchema`, `buildServiceSchema`, `buildItemListSchema`, `buildArticleSchema`, `buildPricedServiceSchema`). **Emit inline JSON-LD with `jsonLdScript()`** — under `@unhead/vue` v2 the inline-content key is `innerHTML`, NOT `children` (the v1 key silently renders as an attribute, producing an empty `<script>`).
- Redirects and headers are Cloudflare Pages files in `public/` (`_redirects`, `_headers`), copied verbatim to `dist/`. The 301s from old SEO URLs (e.g. `/chape-beton-ile-de-france`) live in `public/_redirects` — update there when changing section IDs or adding landing URLs. `robots.txt` and `sitemap.xml` are served from `public/`/`dist/` (sitemap is regenerated at build, so don't hand-edit it).

### PWA

Configured in `vite.config.ts` (VitePWA plugin, auto-update). Manifest name is set to "Chape Expert Île-de-France" / short_name "Chape Expert".

### Devis flow (AI quote)

`/devis` is a multi-step form at [src/pages/devis.vue](src/pages/devis.vue) that posts JSON to the Cloudflare Pages Function [functions/api/submit-quote.ts](functions/api/submit-quote.ts) (route `/api/submit-quote`, `onRequestPost` handler). Price computation lives in [src/utils/pricing.ts](src/utils/pricing.ts) — edit `SERVICES` there to adjust the grid.

The function:

- Validates the payload and drops bot submissions via a `website` honeypot field (returns a fake success). No rate-limiting / no Netlify Blobs anymore.
- Generates an image via Pollinations.ai (free, no key, `flux` model) — returns a URL, does NOT store the image.
- Sends **two** Resend emails: the lead to `chapeexpert@gmail.com` (with the client photos attached as base64), and a confirmation to the customer (non-fatal if it fails).

Required env var (Cloudflare Pages → Settings → Environment variables): `RESEND_API_KEY`. The `from` address in [functions/api/submit-quote.ts](functions/api/submit-quote.ts) (`devis@polletantoine.com`) must be a domain verified in Resend (DNS records) before sending will work.
